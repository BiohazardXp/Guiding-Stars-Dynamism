## Decision Tree: Do You Need a Developer?

```
START: I want to change something on my website
│
├─ Does it involve TEXT or PHOTOS?
│  │
│  ├─ YES → Go to CMS (/content) ✅
│  │         You can do it yourself!
│  │         No developer needed.
│  │
│  └─ NO → Continue below...
│
├─ Does it involve DESIGN or STYLING?
│  │
│  ├─ YES → Developer needed ❌
│  │         (colors, fonts, layout, spacing)
│  │
│  └─ NO → Continue below...
│
├─ Does it involve STRUCTURE or LAYOUT?
│  │
│  ├─ YES → Developer needed ❌
│  │         (page layout, navigation, sections)
│  │
│  └─ NO → Continue below...
│
├─ Does it involve FEATURES or FUNCTIONALITY?
│  │
│  ├─ YES → Developer needed ❌
│  │         (search, filtering, login, payments)
│  │
│  └─ NO → Continue below...
│
└─ Does it involve NEW PAGES or FORMS?
   │
   ├─ YES → Developer needed ❌
   │         (new pages, new form fields)
   │
   └─ NO → Developer needed ❌
            (something else structural)
```

## Cost Comparison

### Scenario: You Want 10 Updates

#### All CMS Changes (Text & Photos)
```
10 CMS updates × 2 minutes each = 20 minutes
Cost: $0
Developer needed: No
Result: Done today
```

#### 5 CMS Changes + 5 Code Changes
```
5 CMS updates × 2 min = 10 min (Cost: $0)
5 Code updates × 3 hours = 15 hours (Cost: $1500-2500)
─────────────────────────────
Total: 15+ hours, $1500-2500, 3-5 days
```

**Moral:** Use CMS for what it can do!

## Questions & Answers

**Q: Why can't everything be in CMS?**  
A: Some things (structure, features, logic) are too complex and risky to change through CMS. They need developer testing.

**Q: What if I want to add more CMS fields?**  
A: Developer can add any text field to CMS in ~30 minutes. Cost: $50-100 per field.

**Q: When can I change colors via CMS?**  
A: Phase 3+ (future). Currently requires code change.

**Q: Can I change the layout via CMS?**  
A: Not yet. Phase 3 will include page builder for layout changes.

**Q: Why isn't Phase 2 ready now?**  
A: Prioritized text/photo management first. Phase 2 starts after client feedback.

**Q: If I hire a developer, what's easiest to add?**  
A: New CMS fields (cheapest), then small code changes, then new features.
